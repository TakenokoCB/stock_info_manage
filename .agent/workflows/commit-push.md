# コミット & プッシュ ワークフロー

変更をコミットしてリモートにプッシュする手順です。

## 手順

// turbo-all

1. ステージング
```bash
git add -A
```

2. コミット（変更内容に応じたメッセージを自動生成）
```bash
git commit -m "<適切なコミットメッセージ>"
```

3. プッシュ
```bash
git push
```

## コミットメッセージの規則

- `feat:` 新機能追加
- `fix:` バグ修正
- `docs:` ドキュメント変更
- `style:` コードスタイル変更
- `refactor:` リファクタリング
- `chore:` ビルド・設定変更
