# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

セキュリティの脆弱性を発見した場合は、以下の手順でご報告ください。

### 報告方法

1. **公開しないでください** - セキュリティ問題はパブリックなIssueに投稿しないでください
2. **メールで連絡** - takenoko.cb@gmail.com までご連絡ください
3. **詳細を記載** - 脆弱性の内容、再現手順、影響範囲をできるだけ詳しく記載してください

### 対応フロー

- **24時間以内**: 報告の受領確認
- **72時間以内**: 初期調査と影響評価
- **1週間以内**: 修正計画の策定と共有
- **修正後**: セキュリティアドバイザリの公開

## セキュリティに関するベストプラクティス

### 機密データの取り扱い

このプロジェクトでは以下のデータを機密として扱います：

| データ種別 | 保存場所 | Git |
|------------|----------|-----|
| 保有資産データ | `data/portfolioData.ts` | ❌ 除外 |
| 証券会社CSV | `data/original/` | ❌ 除外 |
| APIキー | `.env` / `.env.local` | ❌ 除外 |

### 開発者向けガイドライン

1. **環境変数を使用** - APIキーやシークレットはコードにハードコードしない
2. **`.gitignore`を確認** - コミット前に機密ファイルが含まれていないか確認
3. **依存関係を更新** - `npm audit` を定期的に実行し、脆弱性を修正

```bash
# 脆弱性チェック
npm audit

# 自動修正
npm audit fix
```
